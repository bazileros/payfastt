`@bazileros/payfast` integrates the [PayFast](https://www.payfast.co.za) South African payment gateway into your Convex app. It handles the two PayFast integration modes and processes ITN webhooks automatically.

## Integration modes

The component supports both of PayFast's integration modes:

| Mode | Use case | How it works |
|---|---|---|
| **Custom Integration** | One-time payments, subscription signup | Generate signed HTML form → redirect buyer to PayFast → return via return/cancel URL |
| **REST API** | Subscription management, refunds, tokenized charges | Server-to-server authenticated requests using header signing |

## Design decisions

<Accordion title="Why ctx.env instead of process.env?">
  Convex components declare required env vars in `defineComponent({ env: {...} })` using Zod-style validators. This is enforced at deploy time — Convex prompts the developer for every declared var. Access is via `ctx.env.PAYFAST_MERCHANT_ID`, providing full type safety. The old approach (`process.env`) caused silent checkout failures when a passphrase was missing.
</Accordion>

<Accordion title="Why are React hooks parameter-based instead of using a Provider?">
  Convex components are accessed via the consumer's `_generated/api` references (`components.payfast`). A Provider pattern would hide this dependency and make the hooks harder to use in non-React contexts. Parameter-based hooks are explicit, testable, and work with any React rendering pattern.
</Accordion>

<Accordion title="Why is sandbox a constructor option and also an env var?">
  The global sandbox mode is set via `PAYFAST_SANDBOX` in Convex env (applies to server-side operations like REST API calls). The `Payfast` class constructor accepts `{ sandbox: true }` as an override, useful for testing specific requests against sandbox while the rest of the app runs in production.
</Accordion>

## Quick start

<Steps>
  <Step>
    Install the package

    ```bash
    npm install @bazileros/payfast
    pnpm add @bazileros/payfast
    bun add @bazileros/payfast
    ```
  </Step>
  <Step>
    Register the component in your Convex backend

    ```ts title="convex/convex.config.ts"
    import { defineApp } from "convex/server";
    import payfast from "@bazileros/payfast/convex.config";

    const app = defineApp();
    app.use(payfast);
    export default app;
    ```
  </Step>
  <Step>
    Set environment variables via Convex CLI or dashboard

    ```bash
    npx convex env set PAYFAST_MERCHANT_ID your_merchant_id
    npx convex env set PAYFAST_MERCHANT_KEY your_merchant_key
    npx convex env set PAYFAST_PASSPHRASE your_passphrase
    npx convex env set PAYFAST_SANDBOX true
    ```
  </Step>
  <Step>
    See the [Installation guide](/docs/payfast/installation) for the ITN webhook setup and full walkthrough.
  </Step>
</Steps>
