This guide walks through adding the PayFast component to a Convex backend, configuring environment variables, and mounting the ITN webhook.

## 1. Install the package

<Tabs>
  <Tab label="npm">
    ```bash
    npm install @bazileros/payfast
    ```
  </Tab>
  <Tab label="pnpm">
    ```bash
    pnpm add @bazileros/payfast
    ```
  </Tab>
  <Tab label="bun">
    ```bash
    bun add @bazileros/payfast
    ```
  </Tab>
</Tabs>

<Callout type="info">
  `@bazileros/payfast` has a peer dependency on `convex ^1.36.1` and `react ^18.3.1 || ^19.0.0`. If you're not using React, the React peer dep is optional.
</Callout>

## 2. Register the component

In your backend's Convex app configuration:

```ts title="convex/convex.config.ts"
import { defineApp } from "convex/server";
import payfast from "@bazileros/payfast/convex.config";

const app = defineApp();
app.use(payfast);
export default app;
```

<Callout type="warning">
  The import from `@bazileros/payfast/convex.config` is a separate entrypoint from the main package. It must resolve to the component definition, not a client file. The package's `exports` map handles this — just use the exact path shown above.
</Callout>

### What this does

`app.use(payfast)` registers the component's tables and functions in your Convex deployment. After registration:

- Four new tables are created: `payfast_transactions`, `payfast_subscriptions`, `userProfiles`, `payfast_itnLogs`
- The component's queries, mutations, and actions are available at `components.payfast.lib.*`
- The component's schema (env vars, validators) becomes part of your deployment's contract

Once registered, run codegen to update your generated API:

```bash
npx convex codegen
```

This regenerates `convex/_generated/api.ts` to include `components.payfast` with full type information.

## 3. Set environment variables

The component declares four required environment variables:

```ts title="packages/payfast/src/component/convex.config.ts"
export default defineComponent("payfast", {
  env: {
    PAYFAST_MERCHANT_ID: v.string(),
    PAYFAST_MERCHANT_KEY: v.string(),
    PAYFAST_PASSPHRASE: v.string(),
    PAYFAST_SANDBOX: v.string(),
  },
});
```

<Callout type="warning">
  All four are declared with `v.string()` — even `PAYFAST_SANDBOX`. Convex validates that the value is a string at deploy time. Use `"true"` (string) for sandbox, not `true` (boolean).
</Callout>

### Set via CLI

```bash
npx convex env set PAYFAST_MERCHANT_ID    12345678
npx convex env set PAYFAST_MERCHANT_KEY   abc123abc123abc1
npx convex env set PAYFAST_PASSPHRASE     your-secret-passphrase
npx convex env set PAYFAST_SANDBOX        true
```

### Set via Convex dashboard

Navigate to your project on [dashboard.convex.dev](https://dashboard.convex.dev) → Settings → Environment Variables.

<Callout type="tip">
  For local development, you can also use Convex's local dev environment with `npx convex dev`. Convex prompts you for env vars during initial setup.
</Callout>

### Where to get PayFast credentials

1. Log in to [sandbox.payfast.co.za](https://sandbox.payfast.co.za) (test) or [www.payfast.co.za](https://www.payfast.co.za) (production)
2. Go to **Settings → API & Credentials**
3. Find your **Merchant ID**, **Merchant Key**, and set a **Passphrase** (if not already set)
4. The passphrase is used for signature generation — keep it secret

## 4. Mount the ITN webhook

The ITN webhook is how PayFast notifies your app about payment events. Mount it in your Convex HTTP router:

```ts title="convex/http.ts"
import { httpRouter } from "convex/server";
import { components } from "./_generated/api";
import { registerRoutes } from "@bazileros/payfast/http";

const http = httpRouter();

registerRoutes(http, components.payfast);

export default http;
```

### With event handlers

```ts title="convex/http.ts"
import { httpRouter } from "convex/server";
import { components } from "./_generated/api";
import { registerRoutes } from "@bazileros/payfast/http";

const http = httpRouter();

registerRoutes(http, components.payfast, {
  events: {
    onAny: async (ctx, pfData) => {
      console.log("ITN received:", pfData.pf_payment_id);
    },
    onPaymentComplete: async (ctx, pfData) => {
      await ctx.runMutation(api.myMutations.grantAccess, {
        pfPaymentId: pfData.pf_payment_id,
        amount: pfData.amount_gross,
        email: pfData.email_address,
      });
    },
    onPaymentCancelled: async (ctx, pfData) => {
      await ctx.runMutation(api.myMutations.releaseHold, {
        pfPaymentId: pfData.pf_payment_id,
      });
    },
    onPaymentRefunded: async (ctx, pfData) => {
      await ctx.runMutation(api.myMutations.updateLedger, {
        pfPaymentId: pfData.pf_payment_id,
      });
    },
    onSubscriptionCancelled: async (ctx, pfData) => {
      await ctx.runMutation(api.myMutations.revokeAccess, {
        token: pfData.token,
      });
    },
    onSubscriptionUpdated: async (ctx, pfData) => {
      await ctx.runMutation(api.myMutations.syncSubscription, {
        token: pfData.token,
        status: pfData.subscription_status,
      });
    },
  },
});

export default http;
```

<Callout type="info">
  The event handlers run **after** the component has persisted the transaction to the database. You can safely call `runQuery` to check state or `runMutation` to update your app's data.
</Callout>

### Custom ITN path

By default the webhook is mounted at `/payfast/itn`. Customize it:

```ts title="convex/http.ts"
registerRoutes(http, components.payfast, {
  itnPath: "/webhooks/payfast/itn",
  events: { ... },
});
```

Then configure PayFast to send ITNs to `https://your-domain.convex.site/webhooks/payfast/itn`.

## 5. Configure PayFast notify URL

In your PayFast dashboard, go to **Settings → Integration** and set the **Notify URL**:

```
https://YOUR_PROJECT.convex.site/payfast/itn
```

<Callout type="warning">
  PayFast sends ITNs to this URL for **every transaction**. The URL must be publicly accessible over HTTPS. Convex HTTP endpoints are always HTTPS.
</Callout>

## 6. Verify the setup

Run the Convex dev server:

```bash
npx convex dev
```

You should see output indicating the component is registered. You can then call a health check:

```ts title="convex/healthCheck.ts"
import { query } from "./_generated/server";

export const get = query({
  handler: async () => "OK",
});
```

### Troubleshooting

<Accordion title="Module not found: @bazileros/payfast/convex.config">
  Run `npx convex codegen` after installing the package. The generated `_generated/api.ts` needs to include the component reference. Also verify the package is in your `dependencies` (not `devDependencies`).
</Accordion>

<Accordion title="ITN returns 404">
  Check that `registerRoutes` is called in your `convex/http.ts` and the `itnPath` matches the URL in your PayFast dashboard. Run `npx convex dev` to see the registered routes in the console.
</Accordion>

<Accordion title="Signature mismatch">
  Verify your `PAYFAST_PASSPHRASE` matches exactly what's set in the PayFast dashboard. The passphrase is case-sensitive. If you changed it in PayFast, update the Convex env var and restart the dev server.
</Accordion>

<Accordion title="Convex prompts for env vars">
  This is expected on first deploy. Enter the values from your PayFast dashboard. You can update them later via `npx convex env set` or the Convex dashboard.
</Accordion>
